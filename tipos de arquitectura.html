Tipos de Arquitectura

Arquitectura Monolítica

Es un modelo de desarrollo de software tradicional que utiliza un código base para realizar varias funciones empresariales. Todos los componentes de software de un sistema monolítico son interdependientes debido a los mecanismos de intercambio de datos dentro del sistema.

Ventajas y Desventajas

VENTAJAS

Estructura sencilla y fácil de entender para los desarrolladores.


Al estar en un solo código, es más fácil probar e identificar errores.


No requiere comunicación por red entre componentes, lo que reduce tiempos de respuesta.


Toda la aplicación se despliega en un solo proceso.


No requiere una infraestructura compleja ni herramientas adicionales.

Desventajas

No se pueden escalar componentes de manera independiente, se requiere aumentar la capacidad del servidor completo.


Cualquier cambio en el código puede afectar a toda la aplicación.


Cada actualización implica reinstalar toda la aplicación.


Un error en una parte del código puede hacer que toda la aplicación deje de funcionar.


La gestión del código se vuelve compleja con múltiples desarrolladores trabajando en el mismo proyecto.

Arquitectura Basada en Microservicios

Es un enfoque de desarrollo de software que descompone una aplicación en una colección de servicios pequeños e independientes que se comunican entre sí a través de APIs. Cada servicio se centra en una funcionalidad específica y puede desarrollarse, implementarse y escalar de manera autónoma.

VENTAJAS

Cada microservicio se puede escalar de forma autónoma según la demanda.


Se pueden usar distintos lenguajes de programación y bases de datos para cada microservicio.


Las actualizaciones y correcciones se pueden aplicar a un servicio sin afectar a los demás.


Los equipos pueden trabajar en diferentes microservicios sin interferirse.


Si un microservicio falla, el resto de la aplicación puede seguir funcionando.

Desventajas

No se pueden escalar componentes de manera independiente, se requiere aumentar la capacidad del servidor completo.


Cualquier cambio en el código puede afectar a toda la aplicación.


Cada actualización implica reinstalar toda la aplicación.


Un error en una parte del código puede hacer que toda la aplicación deje de funcionar.


La gestión del código se vuelve compleja con múltiples desarrolladores trabajando en el mismo proyecto.

Arquitectura Serverless

Es un modelo de desarrollo en el que las aplicaciones se ejecutan en la nube sin necesidad de administrar servidores. En lugar de aprovisionar y gestionar infraestructura, los desarrolladores utilizan servicios en la nube que ejecutan el código sólo cuando es necesario.

VENTAJAS

Cada microservicio se puede escalar de forma autónoma según la demanda.


Se pueden usar distintos lenguajes de programación y bases de datos para cada microservicio.


Las actualizaciones y correcciones se pueden aplicar a un servicio sin afectar a los demás.


Los equipos pueden trabajar en diferentes microservicios sin interferirse.


Si un microservicio falla, el resto de la aplicación puede seguir funcionando.
